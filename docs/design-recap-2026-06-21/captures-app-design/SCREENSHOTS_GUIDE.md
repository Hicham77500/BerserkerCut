# Screenshots iOS - Guide de Capture\n\n## 📱 Prochaines Étapes\n\nUne fois les prototypes web validés, procéder aux captures écran réelles de l'app iOS.\n\n## Convention de Nommage\n\nCréer un sous-dossier `screenshots/` et placer les captures selon l'ordre du parcours utilisateur:\n\n```\nscreenshots/\n├── 01-auth-login.png\n├── 02-auth-signup.png\n├── 03-auth-demo-toggle.png\n├── 04-onboarding-step1-identity.png\n├── 05-onboarding-step2-health.png\n├── 06-onboarding-step3-training.png\n├── 07-onboarding-step4-supplements.png\n├── 08-dashboard-home.png\n├── 09-nutrition-overview.png\n├── 10-nutrition-meal-edit.png\n├── 11-nutrition-objectives.png\n├── 12-training-today.png\n├── 13-training-weekly-plan.png\n├── 14-agenda-calendar.png\n├── 15-settings-theme.png\n├── 16-settings-notifications.png\n└── 17-settings-privacy.png\n```\n\n## 📐 Recommandations\n\n- **Résolution:** Full-screen viewport (device default)\n- **Variantes:** Ajouter suffixe `-dark` et `-light` si applicable\n- **Data:** Utiliser profil fictif/demo, aucune info personnelle réelle\n- **Format:** PNG optimisé\n- **Annotations:** Optionnel - flèches/callouts pour CTA clés\n\n## ✅ Checklist\n\n- [ ] Prototypes HTML validés (index.html)\n- [ ] Theme dark implémenté dans l'app iOS (DESIGN.md)\n- [ ] Screenshots iOS full flow (login → dashboard)\n- [ ] Variantes light mode capturées\n- [ ] Annotations UX ajoutées si nécessaire\n- [ ] Validation design vs prototypes\n"
